# OLMo-2-0425-1B

## Introduction

OLMo-2-0425-1B is a 1-billion-parameter autoregressive language model developed by the Allen Institute for AI (AllenAI), released in April 2024 as part of the OLMo 2 open-source model series. As an optimized lightweight variant of the OLMo family, this model is designed for efficient deployment on resource-constrained hardware (including Ascend NPUs) while maintaining competitive performance on general-purpose natural language processing tasks.  

Built on the decoder-only Transformer architecture, OLMo-2-0425-1B inherits the core design principles of the OLMo series—full openness, reproducibility, and ease of use—while introducing targeted optimizations for small-scale models: refined attention mechanisms to reduce memory overhead, a curated multi-domain training corpus to improve generalization, and full compatibility with mainstream LLM frameworks (including Hugging Face Transformers and vLLM). These optimizations make it well-suited for research experiments, lightweight application development, and open-source contributions, particularly in scenarios requiring low-latency inference on Ascend hardware.

## Key Features

| Feature | Description |
| --------- | ------------- |
| **Architecture** | 1B parameter decoder-only transformer (Olmo2ForCausalLM) |
| **Context Window** | Up to 4,096 tokens (native support for OLMo-2 series) |
| **Precision** | Native tensor type: F32; BF16 (BFloat16) optimized for Ascend NPU inference (specified via --dtype bfloat16) |
| **Training Paradigm** | Multi-domain corpus training: 4 Trillion tokens (OLMo-mix-1124 dataset) + 50B tokens (Dolmino-Mix-1124 mid-training) with grouped attention optimization |
| **Specialization** | General-purpose text generation, research experiments, lightweight NLP application development |
| **Deployment** | Optimized for vLLM Ascend with single-NPU deployment (no tensor parallelism required; tensor_parallel_size=1 recommended) |

## Deployment

### Single-Node Deployment

Create a deployment script (`deploy_OLMo-2-0425-1B.sh`):

```shell
#!/bin/sh
export HF_HOME=/data/huggingface_home
export HF_ENDPOINT=https://hf-mirror.com
export ASCEND_RT_VISIBLE_DEVICES=0  # Single NPU card
export MODEL_PATH="allenai/OLMo-2-0425-1B"

vllm serve ${MODEL_PATH} \
    --host 0.0.0.0 \
    --port 8000 \
    --served-model-name allenai/OLMo-2-0425-1B \
    --tensor-parallel-size 1 \  # No parallelism needed for 1B model
    --trust-remote-code \  # Mandatory for custom architecture
    --dtype bfloat16 \  # Optimized for Ascend NPU
    --max-model-len 4096 \
    --gpu-memory-utilization 0.85  # Balance between performance and stability
```

**Configuration Notes**:

- Adjust `ASCEND_RT_VISIBLE_DEVICES` to match your available NPU devices
- Reduce `--max-model-len` if encountering memory constraints
- Keep `--max-model-len=4096` (native context window, no need to modify)

## Performance

### Using vLLM Benchmark

Run the following script to test throughput and latency:

```shell
export VLLM_USE_MODELSCOPE=true
python -m vllm.entrypoints.benchmark_throughput \
  --model allenai/OLMo-2-0425-1B \
  --trust-remote-code \
  
  --dtype bfloat16 \
  --dataset-name random \
  --random-input 200 \
  --num-prompts 200 \
  --request-rate 1 \
  --save-result \
  --result-dir ./perf_results/
```

### Typical Performance Indicators (Atlas A2 Series)

- Throughput: 30-50 tokens/sec (single card)  
- Latency (TPOT): <100ms (for 200-token input, 50-token output)  
- Memory Utilization: ~2.5-3GB (BF16 precision)  

## Troubleshooting

| Issue | Cause | Solution |
| --- | --- | --- |
| **Model Loading Failure** | Missing custom architecture code | Add `--trust-remote-code` flag; ensure `transformers≥4.48` |
| **Blank Output** | Ascend Graph compilation compatibility issue | Add `--compilation-config '{"cudagraph_mode":"FULL_DECODE_ONLY"}'` or switch to `--dtype float16` |
| **Slow Download** | Network restriction | Use domestic mirror (HF-Mirror) as shown in "Model Weight" section |
| **Version Mismatch Error** | Incompatible vLLM version | Install vLLM from main branch: `pip install git+https://github.com/vllm-project/vllm.git` |

## Limitations

1. Language Bias: Training data has a high English proportion, leading to degraded performance on non-English tasks.  
2. Complex Reasoning: Limited by 1B parameter scale, performance on advanced mathematical reasoning and long-text logical chains is weaker than larger models.  
3. Factuality Risk: May generate inaccurate or one-sided information; always verify critical content manually.
